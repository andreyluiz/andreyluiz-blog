ES 2015 - Um pouco sobre variáveis

24/01/2017
  • javascript
  • variaveis

Sabe qual a diferença entre var, let e const? Eu te explico.

var

Quando declaramos uma variável com var, o interpretador JavaScript, não importa a versão, hasteia as variáveis para o topo da função. Isso é conhecido como hoisting entre a comunidade de desenvolvedores. O motivo é porque, na ES 5, a única coisa que provê escopo às variáveis declaradas com var são as funções.

Por causa desse comportamento é sempre recomendado que as variáveis declaradas com var fiquem no topo da função.

Mas e se eu alterar esse código para isso?

Como esperado, o console.log no diz undefined e é fácil perceber o porque. Repare que a variável unknown foi declarada dentro da expressão de função atribuída à variável isUnknown. Portanto, a função sayHello não tem nenhuma referência dessa variável porque o escopo da variável name é outro.

Repare, porém, que a função isUnknown consegue acessar a variável name da função de fora. Isso é conhecido como closure, e certamente criarei um post só para tratar isso, pois é um conceito um pouco mais avançado. Se estiver curioso, veja aqui.

let

Quais os benefícios de declarar variáveis com let?

Primeiramente, variáveis declaradas com essa keyword, preservam escopo de bloco. Ou seja, seu escopo pertencerá à delimitação de chaves ({}) onde ela está contida.

Qual devo usar?

Vai muito do caso e do código no qual está trabalhando. Mas provavelmente você deve usar const sempre que possível. Isso tornará seu código mais seguro, principalmente se estiver trabahando com bibliotecas de terceiros ou código de outras pessoas de onde trabalha.

Mas o uso de qualquer uma das declarações é válido e nenhuma delas é desencorajada.
